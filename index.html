Program Purpose

The Compute Demand Mitigation Benefits Program provides temporary relief payments to applicants whose software-adjacent activity has been determined to create disproportionate load on inference providers, CI systems, observability vendors, and the patience of senior engineers.

Approved participants agree to a cooling period during which they refrain from opening an IDE, requesting agentic continuation, or initiating a repo with the words v2, final, or platform.

Eligibility Guidelines

  • Prompt Dependency Index greater than 0.72.
  • Unreviewed Lines-of-Code Ratio above the current household median.
  • Documented habit of saying “the agent can fix this later.”
  • Ability to provide Alternative Output Evidence in lieu of commits.

Frequently Avoided Questions

Guidance for edge cases

May I write YAML during the benefit period?

Only under licensed exception. Applicants must demonstrate that the YAML cannot schedule a container, create a Jira ticket, or summon a dashboard.

Is pseudocode considered code?

Yes, if a sufficiently motivated agent could turn it into a pull request before lunch.

Can I ask an AI to explain code?

Explanation is permitted. “Now implement the above” is a reportable relapse event.

What if my employer says this is critical?

The Bureau recognizes all dashboards as critical until 90 days after they are forgotten.

Can I attend standup?

Yes, provided no action items are accepted and the phrase “I can take a look” is not used.

What happens if my application is denied?

You may submit Form REGRET-2 within 14 days or create one more prototype, whichever happens first.

Administrative Review

Appeals and Relapse Events

Denied applicants may request reconsideration by demonstrating a compliant non-production posture for two consecutive weekends. Relapse events include installing a package manager, naming a folder scratch, or telling a model to “make it production ready.”
